# PART 2: DASHBOARD & PAYMENT DATA POPULATION FIXES

**Date:** 2026-03-26

**Status:** ✅ All Issues Fixed

---

## Problems Identified

### 🔴 CRITICAL ISSUE #1: Wrong Ticket Status Checks
**File:** `api/stats/get-dashboard-stats.php`  
**Lines:** 49, 86, 105  
**Problem:**
```php
// WRONG - tickets table status is ('valid', 'used', 'cancelled'), NOT 'paid'
WHERE t.status = 'paid'

// Tickets NEVER have status='paid', they only have 'valid', 'used', 'cancelled'
// Payment status is stored in PAYMENTS table, not TICKETS table
```

**Impact:** All dashboard stats returned 0 because queries returned empty result sets  
**Root Cause:** Schema misunderstanding - ticket status vs payment status stored in different tables

---

### 🔴 CRITICAL ISSUE #2: Invalid Database Column
**File:** `api/stats/get-dashboard-stats.php`  
**Lines:** 58, 89  
**Problem:**
```php
// WRONG - users table does NOT have 'is_active' column
WHERE is_active = 1

// 'is_active' is in auth_accounts table, not users table
// users table has: deleted_at (NULL = active)
```

**Impact:** Query would fail silently or return 0 users  
**Root Cause:** Incorrect schema reference - active status stored in related table

---

### 🟠 ISSUE #3: Variable Scope Bug in Payment Processing
**File:** `api/payments/verify-payment.php`  
**Lines:** 205, 226-227  
**Problem:**
```php
$ticket_id = null; // clear for transaction safety (line 205)

// Later, trying to use it:
$pdo->prepare("UPDATE tickets SET qr_code_path = ? WHERE id = ?")
    ->execute([..., $ticket_id]); // NULL! Query won't match any rows
```

**Impact:** QR code paths not updated for tickets after payment verification  
**Root Cause:** Duplicate code from loop being used after loop ended

---

## Solutions Applied

### Fix #1: Update get-dashboard-stats.php - Client Tickets
```php
// BEFORE:
WHERE t.status = 'paid'

// AFTER:
INNER JOIN payments p ON t.payment_id = p.id
WHERE p.status = 'paid'
```

### Fix #2: Update get-dashboard-stats.php - Active Users
```php
// BEFORE:
FROM users WHERE is_active = 1

// AFTER:
FROM users u
INNER JOIN auth_accounts a ON u.user_auth_id = a.id
WHERE a.is_active = 1 AND u.deleted_at IS NULL
```

### Fix #3: Update get-dashboard-stats.php - Admin Tickets
```php
// BEFORE:
FROM tickets WHERE status = 'paid'

// AFTER:
FROM tickets t
INNER JOIN payments p ON t.payment_id = p.id
WHERE p.status = 'paid'
```

### Fix #4: Update get-dashboard-stats.php - User Tickets
```php
// BEFORE:
FROM tickets WHERE user_id = ? AND status = 'paid'

// AFTER:
FROM tickets t
INNER JOIN payments p ON t.payment_id = p.id
WHERE t.user_id = ? AND p.status = 'paid'
```

### Fix #5: Refactor verify-payment.php - Eliminate Duplicate Code
```php
// Removed duplicate code section that was trying to update with NULL $ticket_id
// Removed lines 205-227 duplicate "Generate QR & PDF" block
// QR codes are now generated only once inside the loop (line 196-200)
// Added ticket_ids array to track all created ticket IDs
```
